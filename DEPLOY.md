# 小程序部署指南

## 📱 微信小程序部署步骤

### 1. 编译项目

确保开发服务器正在运行，或者执行生产构建：

```bash
# 开发模式（带热更新）
npm run dev:weapp

# 生产模式（压缩优化）
NODE_ENV=production npm run build:weapp
```

编译完成后，会在 `dist` 目录生成小程序文件。

### 2. 打开微信开发者工具

1. 下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 打开微信开发者工具
3. 选择"小程序项目"
4. 点击"+"创建新项目

### 3. 导入项目

在创建项目对话框中：

- **项目目录**：选择 `/Users/linleil/Downloads/mini/badminton-score-miniapp/dist`
- **AppID**：
  - 如果有小程序账号，填入你的 AppID
  - 如果只是测试，可以选择"测试号"
- **项目名称**：羽毛球计分器
- **开发模式**：小程序
- **后端服务**：不使用云服务

点击"确定"创建项目。

### 4. 配置项目信息

在微信开发者工具中：

1. 点击右侧"详情"按钮
2. 在"本地设置"中确认：
   - ✅ 使用 npm 模块（本项目不需要，因为 Taro 已处理）
   - ✅ ES6 转 ES5（建议勾选）
   - ✅ 增强编译（建议勾选）

3. 在"域名信息"中（如果需要网络请求）：
   - 配置合法的 request 域名
   - 本项目暂不需要网络请求

### 5. 预览和调试

1. 在模拟器中查看小程序效果
2. 点击顶部"预览"按钮，生成二维码
3. 使用微信扫码在真机上测试

### 6. 上传代码

1. 点击右上角"上传"按钮
2. 填写版本号（如 1.0.0）
3. 填写版本说明
4. 点击"上传"

### 7. 提交审核

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入"管理" -> "版本管理"
3. 找到刚上传的版本，点击"提交审核"
4. 填写审核信息：
   - 功能页面：列出所有功能页面
   - 测试账号：如需要登录，提供测试账号
   - 补充说明：说明小程序用途

### 8. 发布上线

审核通过后：
1. 在"版本管理"中找到审核通过的版本
2. 点击"发布"
3. 小程序即可在所有用户端看到

## 🔧 常见问题

### 1. 编译错误

如果遇到编译错误，尝试：

```bash
# 清理缓存
rm -rf dist node_modules/.cache

# 重新安装依赖
npm install

# 重新编译
npm run build:weapp
```

### 2. 样式问题

- 确保使用 rpx 单位而不是 px
- 避免使用小程序不支持的 CSS 特性
- 检查是否有未闭合的标签

### 3. 路由跳转失败

- 检查 `app.config.ts` 中的页面路径是否正确
- 确保页面文件存在且命名正确
- 路径必须以 `/pages/` 开头

### 4. 真机调试

在微信开发者工具中：
1. 点击"预览"生成二维码
2. 用手机微信扫码
3. 在手机微信中打开"开发调试"
4. 可以查看控制台日志和网络请求

## 📊 性能优化建议

### 1. 图片优化
- 使用合适的图片格式（WebP 优先）
- 压缩图片大小
- 使用 CDN 加载图片

### 2. 代码分包
如果项目变大，考虑使用分包加载：

```typescript
// app.config.ts
export default defineAppConfig({
  pages: ['pages/home/index'],
  subPackages: [
    {
      root: 'sub',
      pages: ['pages/config/index', 'pages/match/index']
    }
  ]
})
```

### 3. 减少 setData 调用
- 避免频繁更新状态
- 合并多次 setState 调用
- 只更新必要的数据
